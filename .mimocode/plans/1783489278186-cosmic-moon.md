# 迭代管理优化计划

## 需求概述
1. **状态排序优化**：按状态优先级排序，去掉"代码审查"状态，记录状态变更时间
2. **本地目录导入**：每次可指定不同目录，记录最近一次导入目录，双向同步
3. **Markdown预览美化**：优化markdown渲染样式

---

## 1. 状态排序与时间记录

### 1.1 修改后端实体 `Iteration.java`
```java
// 新增字段
@Column(name = "status_changed_at")
private LocalDateTime statusChangedAt;  // 状态最后变更时间

@Column(name = "completed_at")
private LocalDateTime completedAt;       // 完成时间
```

### 1.2 修改前端类型 `types/iteration.ts`
- 移除 `CODE_REVIEW` 状态
- 更新状态列表：`TODO`(待开发) → `IN_PROGRESS`(开发中) → `TESTING`(测试中) → `DONE`(已完成)
- Iteration 接口添加 `statusChangedAt` 和 `completedAt` 字段

### 1.3 排序逻辑
- 后端默认排序：先按 status 排序（权重：TODO=1, IN_PROGRESS=2, TESTING=3, DONE=4），再按 id 降序

### 1.4 Service 层改动
- `updateIteration` 检测状态变更，自动记录 `statusChangedAt`
- 状态变为 DONE 时自动记录 `completedAt`

## 2. 本地目录导入与双向同步

### 2.1 新增后端实体 `IterationImportConfig.java`
```java
@Entity
@Table(name = "iteration_import_config")
public class IterationImportConfig {
    @Id @GeneratedValue
    private Long id;
    private String baseDirPath;  // 最近一次导入的目录路径
    private Boolean active;
    private LocalDateTime createdAt;
    private LocalDateTime updatedAt;
}
```

### 2.2 新增 API 接口
| 接口 | 说明 |
|------|------|
| `POST /api/iterations/import` | 从指定目录导入需求 |
| `GET /api/iterations/import/config` | 获取最近导入配置 |
| `POST /api/iterations/{id}/sync` | 同步单个需求到本地目录 |
| `POST /api/iterations/sync-all` | 同步所有需求到本地目录 |

### 2.3 导入逻辑
- 扫描指定目录下的子文件夹
- 文件夹名格式：`#1234-项目简称-需求标题`
- 文件夹内 `.txt` → 开发笔记(developmentNotes)
- 文件夹内 `.md` → 发布文档(releaseNotes)
- 保存导入目录路径到 `IterationImportConfig`

### 2.4 双向同步机制
**页面创建/编辑 → 本地文件：**
- 创建需求时，如果已配置导入目录，自动创建文件夹
- 编辑需求时，自动更新对应的 txt/md 文件

**本地文件 → 页面数据：**
- 导入时读取本地文件内容到数据库
- 可手动触发重新同步

### 2.5 文件命名格式
```
导入目录/
├── #1234-order-service-订单服务优化/
│   ├── notes.txt      (开发笔记)
│   └── release.md     (发布文档)
├── #1235-user-center-用户中心重构/
│   ├── notes.txt
│   └── release.md
```
