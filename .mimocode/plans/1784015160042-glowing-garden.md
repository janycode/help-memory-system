# Plan: Exclude Hidden Directories (starting with `.`) from Iteration Sync

## Problem
When syncing/importing iterations from a local directory, hidden directories (like `.obsidian`) are being treated as iteration folders and imported as new iterations. Need to filter out directories whose names start with `.`.

## Root Cause
`IterationImportService.java` has 3 scanning entry points that iterate over subdirectories and pass them to `parseFolderToIteration()`. None of them filter out hidden directories:

1. **`importFromDirectory()`** (line 57) — initial import from directory
2. **`autoSync()`** (line 293) — bidirectional auto-sync
3. **`checkAndImportNewFolders()`** (line 521) — detect and import new folders

## Fix

### Single file: `backend/src/main/java/com/example/mynewwork/service/IterationImportService.java`

**Add a helper method** to check if a directory should be skipped:

```java
private boolean shouldSkipDirectory(Path entry) {
    String name = entry.getFileName().toString();
    // Skip hidden directories (starting with .)
    return name.startsWith(".");
}
```

**Add the skip check** in all 3 scanning loops, right after `if (Files.isDirectory(entry))`:

1. **`importFromDirectory()`** (line 59): Add `|| shouldSkipDirectory(entry)` to the condition
2. **`autoSync()`** (line 295): Same pattern
3. **`checkAndImportNewFolders()`** (line 523): Same pattern

### Exact changes:

**Line 59** (`importFromDirectory`):
```java
// Before:
if (Files.isDirectory(entry)) {
// After:
if (Files.isDirectory(entry) && !shouldSkipDirectory(entry)) {
```

**Line 295** (`autoSync`):
```java
// Before:
if (Files.isDirectory(entry)) {
// After:
if (Files.isDirectory(entry) && !shouldSkipDirectory(entry)) {
```

**Line 523** (`checkAndImportNewFolders`):
```java
// Before:
if (Files.isDirectory(entry)) {
// After:
if (Files.isDirectory(entry) && !shouldSkipDirectory(entry)) {
```
